🎯 Available Demo Module

A special lazy_demo package is included in this demo. It contains submodules that print messages when imported, making it easy to see when lazy imports are reified:

  • lazy_demo - Main package (prints "📦 lazy_demo package loaded!")
  • lazy_demo.heavy - Heavy computation module (prints "⚙️ lazy_demo.heavy loaded!")
  • lazy_demo.utils - Utility functions (prints "🔧 lazy_demo.utils loaded!")
  • lazy_demo.data - Data module (prints "📊 lazy_demo.data loaded!")

💡 Try: lazy import lazy_demo.heavy then access lazy_demo.heavy.compute() to see the lazy import in action!

About Lazy Imports (PEP 810)

Lazy imports introduce explicit syntax (lazy import module) that defers module loading until first use. The module doesn't appear in sys.modules until you actually access it, significantly improving startup time for applications with many imports.

Note: This is an experimental WebAssembly build of Python. Some features like networking, subprocesses, and threading are not available. For more information, see PEP 810 and Tools/wasm/README.md .

SharedArrayBuffer, which is required for this demo, is not available in your browser environment. One common cause of this failure is loading index.html directly in your browser instead of using server.py as described in Tools/wasm/README.md .

For more details about security requirements for SharedArrayBuffer, see this MDN page.
